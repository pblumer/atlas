# Glossary

Terms used throughout Atlas's documentation and code.

**Behavior** — The type-specific logic for a BPMN element (service task, gateway, timer event, subprocess). Implements `OnActivated`, `OnCompleting`, `OnTerminating`. Dispatched by `BpmnElementType`.

**Boundary event** — An event attached to an activity that can interrupt (or run alongside) it: a timeout, an error, a message. Compiled as a `Span` on the enclosing scope.

**Command** — An intention to do something, submitted to a partition. May be rejected. Not persisted. Turned into events by the processor.

**CompiledProcess** — The immutable, integer-indexed output of the graph compiler for one process version. Read concurrently without locks.

**Detail table** — A type-specific table (e.g. `serviceTasks`, `timers`) that a `CompiledNode` indexes into, keeping nodes small and cache-friendly.

**Element instance** — A single active occurrence of a BPMN element within a process instance. The carrier of a "token". Goes through the activating→activated→completing→completed lifecycle.

**Event** — A fact that something happened. Never rejected. Appended to the log. The only thing persisted. Replayed on recovery.

**FEEL** — Friendly Enough Expression Language, the standard expression language from the DMN spec, used for gateway conditions, mappings, and timer definitions. Compiled ahead of time.

**Followup command** — A command generated by the processor while handling another command (e.g. `SequenceFlowTaken` producing an `ActivateElement`). Enqueued and made durable in the next batch.

**fsync** — The OS call that forces buffered writes to physical disk. The unit of durability. Expensive, hence group commit.

**Group commit** — Batching many events and making them durable with a single `fsync`, so throughput scales with batch size rather than disk round-trips.

**Incident** — A first-class state entity raised when a token cannot proceed (job failed past retries, expression error, missing variable). Pauses the token for operator intervention; resolving it resumes execution.

**Intent** — Part of the record discriminator. What is happening or has happened (Activating, Completing, Created, Triggered, ...).

**Job** — A unit of service-task work handed to an external worker. Identified by a job key; completed via a command that flows back through the processor.

**Key** — A globally unique 64-bit identifier for a stateful entity, with the partition encoded in the high 16 bits.

**Linearization** — The compiler stage that flattens BPMN's hierarchy into flat, indexed slices with scope references.

**Partition** — A fully independent execution unit: its own command queue, single-writer processor, WAL, and state store. A process instance lives entirely in one partition.

**Position** — The monotonic sequence number of a record in the log. `SourcePos` references the causing record's position.

**Process definition** — A deployed, compiled BPMN model. Identified by a `ProcessDefKey`; multiple versions can coexist.

**Process instance** — A single running execution of a process definition.

**ProcessingContext** — The narrow API a behavior uses: read state, append events (fact + state mutation), schedule followups and side effects.

**Record** — A log entry: a header plus a typed payload. Either a command, an event, or a command rejection.

**Scope** — An enclosing structural context (the process root, a subprocess, an activity with boundary events). Tracks active-children counts for completion detection.

**Side effect** — Externally-visible work (notifying a worker, sending a response) run *after* the batch's fsync, so it never precedes durability.

**Single writer** — The model where exactly one goroutine processes commands for a partition, eliminating locks and making execution deterministic.

**State applier (`applyToState`)** — The one deterministic function that mutates state from a record. Runs identically live and on recovery, guaranteeing log/state consistency.

**Struct-of-arrays** — The layout where topology (incoming/outgoing flows) lives in shared contiguous arrays indexed by offset+count, rather than per-node slices.

**Token** — Conceptually, the thread of control moving through a BPMN model. In Atlas, represented by active element instances rather than allocated objects.

**ValueType** — Part of the record discriminator. Which kind of entity a record concerns (ElementInstance, Job, Timer, Message, ...).

**WAL (write-ahead log)** — The append-only, segmented log of events. The single source of truth; the state store is its materialization.
